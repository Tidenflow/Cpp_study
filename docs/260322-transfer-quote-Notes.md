> 学习日期：2026年3月22日

---

## 一、三种传参方式的本质
在 C++ 函数调用中，核心问题只有一个：**函数内部到底在操作谁的内存**。

- 传值：操作的是实参的副本（新内存）
- 传指针：通过地址操作实参（原内存）
- 传引用：通过别名操作实参（原内存）

可以把它们看成同一条光谱：

- `T`：给你一份“拷贝”
- `T*`：给你“地址”
- `T&`：给你“原对象的别名”

---

## 二、代码中的对比实验
### 1. 传值（`passByValue(int num)`）
```cpp
void passByValue(int num) {
    num = 100;
}
```

**现象**：函数内修改 `num` 后，外部变量不变。  
**原因**：`num` 是调用时复制出来的局部副本。

---

### 2. 传指针（`passByPointer(int* numPtr)`）
```cpp
void passByPointer(int* numPtr) {
    if (numPtr != nullptr) {
        *numPtr = 200;
    }
}
```

**现象**：函数内通过 `*numPtr` 修改后，外部变量改变。  
**原因**：传入的是地址，解引用后访问的是原对象。  
**额外责任**：必须处理空指针（`nullptr`）问题。

---

### 3. 传引用（`passByReference(int& numRef)`）
```cpp
void passByReference(int& numRef) {
    numRef = 300;
}
```

**现象**：函数内修改后，外部变量改变。  
**原因**：`numRef` 是原对象的别名，不是新对象。  
**特点**：语法像普通变量，但语义是“直接改原值”。

---

## 三、传值、传指针、传引用的关系
### 1. 共同点
- 都是函数拿到实参信息的方式
- 都能实现“把数据传进函数”
- 都可以配合 `const` 形成只读接口

### 2. 关键差异
| 维度 | 传值 `T` | 传指针 `T*` | 传引用 `T&` |
|------|---------|------------|------------|
| 是否拷贝对象 | 是 | 否 | 否 |
| 能否改原对象 | 否 | 能（解引用后） | 能（直接） |
| 是否可为空 | 否 | 是（可 `nullptr`） | 否（必须绑定对象） |
| 调用语法 | 最直观 | 需取地址/解引用 | 最接近传值体验 |
| 安全性负担 | 低 | 高（空指针、悬空指针） | 中低（无空值语义） |

### 3. 一句话理解
- **传值**是“复制一份再改”  
- **传指针**是“拿地址去改”  
- **传引用**是“换个名字直接改”

---

## 四、为什么实际开发更偏爱传引用
### 1. 性能更稳
对于 `std::string`、容器、自定义大对象，传值可能触发拷贝；传引用通常避免不必要拷贝。

### 2. 可读性更好
`func(obj)` 比 `func(&obj)` 更自然，函数体内也不需要反复 `*ptr`。

### 3. 空值风险更低
引用必须绑定到有效对象，天然避免“忘记判空就解引用”的常见错误。

### 4. 与 `const` 组合后非常实用
工程里最常见签名：
```cpp
void print(const std::string& s);
```
它同时做到两件事：
- 不拷贝（高效）
- 不修改实参（安全）

---

## 五、什么时候仍然使用传指针
传指针并没有过时，典型场景仍然合理：

- 需要“可为空”语义（例如参数可选）
- 需要表达所有权或资源关系（尤其配合智能指针）
- 与 C 接口互操作时必须使用指针形式

可见：**引用是默认优先；指针用于明确表达“可能为空”或“指向关系”。**

---

## 六、与源码中 const 示例的连接
源码还演示了：
- `const int*`：不能通过指针改值，但可改指向
- `int* const`：可改值，但不能改指向
- `const int* const`：值和指向都不能改
- `const int&`：只读别名，常用于函数参数

这进一步说明：实际设计接口时，常把“是否允许修改”“是否允许为空”作为首要维度。

---

## 七、实战建议（可直接套用）
- 小型基础类型（如 `int`、`double`）优先传值
- 大对象默认用 `const T&`
- 需要在函数内修改调用者对象时用 `T&`
- 需要表达“参数可缺省/可为空”时用 `T*`

---

## 八、总结
在 `transfer_quote.cpp` 这份代码里，三种传参方式的核心结论是：

- 传值隔离修改，语义安全但可能有拷贝成本
- 传指针与传引用都能改原对象
- 引用在“性能 + 可读性 + 安全性”上更平衡，所以工程里更常作为默认选择

**默认策略可以记成一句话：能用 `const T&` 就先用它；确实需要空值语义再用 `T*`。**
